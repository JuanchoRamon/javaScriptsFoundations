Race case

Your tasks are broken up for you so you can plan your approach and gauge what each aspect of functionality is worth.

PART A: Validation

In this part of the challenge, you are tasked with validating the registrant's age and name. We will keep the validation simple for this assessment. The rules are as follows:

  • The name cannot be an empty string
  • Age is required (it cannot be empty), it cannot be non-numeric, and it cannot be less than 21.

Your validation logic should be handled by a separate function from the registration button's click event. Provide one custom error message per input. You can output confirmations simply in an alert, or for more marks, you can output the results to the document (you will need to add to the HTML for the later method).

PART B: Processing the Registration Costs

Only output the registration cost if you receive valid name and age input.

There are formulas that you will need to apply to determine the registration cost. Here is the logic:

  • 5KM race: 10 + (age/2)
  • 10KM race: 18 + (age/3)

One added detail is this: if the user is a race club member, s/he gets a $5.00 discount.

Output the final registration cost to the user in an alert using the following message format: "Name. your registration cost for the 5KM race is $"
